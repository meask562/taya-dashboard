# Prompts لـ Claude Code — Dashboard متابعة أسعار صفقات محيط مدينة طايا الصناعية (TIC)

> **الغرض:** dashboard مستضاف على **Vercel** يتحدّث يومياً، متجاوب (جوال + كمبيوتر)، بهوية TIC.
> **المعمارية:** الجلب **محلي على ماك المالك عبر Chrome المسجَّل دخوله في بسيطة** + نشر سحابي على Vercel.
> **المدير:** يفتح رابطاً واحداً فقط — لا يدخل شيئاً، لا يجدّد شيئاً.
> **المالك (أبو تركي):** يسجّل دخول بسيطة في Chrome مرة (تذكّرني)، ويجدّده فقط حين تنتهي الجلسة (نادر، إشعار).

---

## ⚠️ اقرأ أولاً — مصدر البيانات وآلية الجلب (لا تتجاوزها)

1. **المصدر:** بسيطة/Paseetah API الداخلي (`paseetah.com/api/precord/rer_transactions/data`) — يغلّف صفقات
   وزارة العدل/السجل العقاري. لا يوجد API رسمي مجاني بلا مصادقة؛ البوابة (OTP/نفاذ) بنيوية في كل القنوات.
2. **آلية تجاوز OTP المتكرر:** الجلب يعمل **على جهاز المالك مستخدماً ملف Chrome الشخصي المسجَّل دخوله**
   (persistent user-data-dir + «تذكّرني») → يرث الجلسة الحيّة، فلا OTP كل تشغيل. OTP فقط عند انتهاء الجلسة.
3. **الجلب محلي، النشر سحابي:** السكربت يجلب ويبني على الماك ثم `git push` → Vercel يعيد النشر → رابط المدير يتحدّث.
4. **التسلسل المرجعي (إلزامي):** بسيطة → REGA/عدل → بلديات. الإعلانات (عقار/Bayut) **سياق فقط**.
5. **الأمانة التحليلية:** كل رقم يُنسب لمصدره وتاريخه؛ صرّح بعدم اليقين؛ لا تختلق صفقات.
6. صفقات السجل تُسجَّل عبر أيام لا لحظياً → الجلب يومي للالتقاط المبكر؛ trend المؤشر أسبوعي/شهري.

**النطاق (كل محيط طايا):**
| المخطط | الوصف | نطاق سعري مرجعي (Dec2025–Jun2026) |
|---|---|---|
| 2377 | صناعي المصفاة الملاصق (قطع 400–750م) | 1,600–2,000 ر.س/م² |
| 3880/1 | الفوزان الصناعية (قطع ~5,000م) | 1,100–2,400 ر.س/م² |

| 3200 | سكن عمالة / سكني مجاور (~600م) | 833–1,200 ر.س/م² |
| (مرجع) | السلي — مبانٍ مستودعات | 2,650–3,500 ر.س/م² |

معرّفات الأحياء في بسيطة: المصفاة 11010192، الصناعية الجديدة 11010065، السلي 11010061، المصانع 11010117 (Riyadh city_id=1, region=1).

**هوية TIC:** navy `#28285D` / `#0F1B3C`، teal `#5EC2C9`، الشعار من `tic-taya.sa/branding/`. عربي RTL.
**تحذير تقني:** أي SVG داخل RTL ثبّت عليه `direction:ltr`؛ وطبّع الأرقام العربية-الهندية قبل التحليل.

## المعمارية المعتمدة (جلب محلي بـ Chrome + نشر Vercel)

```
launchd (مجدول macOS يومي)
   └─> fetch.py (Python + Playwright يفتح Chrome بملف المالك المسجَّل دخوله في بسيطة)
         └─> يستدعي API بسيطة الداخلي (نفس الطريقة المثبتة) → يسحب الصفقات الجديدة
               └─> SQLite محلي (dedup) + يحسب metrics.json
                     └─> git commit + push → Vercel auto-deploy
                           └─> رابط المدير يتحدّث (يفتحه من أي جهاز)

عند انتهاء الجلسة (401): إشعار سطح المكتب + بريد للمالك → يسجّل دخول Chrome مرة → يستأنف تلقائياً
```
- **لماذا محلي للجلب؟** لأن جلسة بسيطة في Chrome المالك تحلّ مشكلة OTP بأقل احتكاك.
- **لماذا سحابي للعرض؟** كي يرى المدير الرابط من أي مكان دون أن يعتمد على تشغيل جهاز المالك لحظة الفتح.
- **شرط:** جهاز المالك شغّال وغير نائم وقت التشغيل المجدول (اختر وقتاً معتاداً أو اضبط إيقاظاً).

## Prompt 1 — جالب بسيطة عبر Chrome profile المسجَّل (محلي)

```
اكتب /fetcher/fetch.py يسحب صفقات السجل العقاري من Paseetah مستخدماً Playwright مع Chrome profile
مسجَّل دخوله مسبقاً (لا OTP داخل السكربت).

التهيئة:
- استخدم playwright.chromium.launch_persistent_context(user_data_dir=PROFILE_PATH, channel="chrome",
  headless=False في أول تشغيل ثم headless=True لاحقاً). PROFILE_PATH من env.
- افتح تبويب paseetah.com للتأكد أن الجلسة حيّة؛ لو ظهرت صفحة تسجيل دخول → ارفع SessionExpiredError.

السحب (نفس النمط المثبت):
- على التبويب المسجَّل، نفّذ fetch داخل الصفحة (same-origin) إلى
  POST /api/precord/rer_transactions/data
  body: {"page":N,"regions":[1],"cities":[1],"neighborhoods":[11010192,11010065,11010061,11010117],
  "planExactMatch":true,"parcelExactMatch":true,"realEstateNumberExactMatch":true,
  "sort_column":"transaction_date","sort_order":"descending"}
  مع X-XSRF-TOKEN من الكوكي و X-Requested-With. 50 صف/صفحة.
- loop على الصفحات حتى transaction_date < cutoff (أول مرة 24 شهراً؛ ثم incremental بعد آخر تاريخ مخزّن).
- النطاق: المخططات 2377، 3880/1، 3796، 3200 + bounding box حول 24.4887N,46.8813E احتياطاً.

التخزين:
- SQLite /data/taya.db جدول deals بمفتاح فريد (real_estate_number+transaction_date+parcel) UPSERT.
  أعمدة: transaction_date, plan, parcel, real_estate_number, area, usage, type, meter_price, amount,
  lat, lng, source='Paseetah/RER', fetched_at.
- طبّع الأرقام العربية-الهندية إلى لاتينية وحوّل الأرقام قبل التخزين.
- صدّر snapshot /data/snapshots/YYYY-MM-DD.json.

عند SessionExpiredError: أرسل إشعار macOS (osascript) + بريد (اختياري) «افتح Chrome وسجّل دخول بسيطة»،
واخرج بكود غير صفري دون كسر آخر بيانات. أضف اختبار وحدة على دالة التطبيع.
اكتب أيضاً تعليمات «تسجيل الدخول لمرة واحدة»: تشغيل fetch.py --login يفتح Chrome مرئياً ليسجّل المالك ويُحفظ الـ profile.
```

## ملاحظات تشغيلية (لإبلاغ المالك والمدير)
