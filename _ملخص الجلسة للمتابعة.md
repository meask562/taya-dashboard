# لوحة طايا (TIC) — ملخّص الجلسة للمتابعة
**آخر تحديث:** 2026-06-20 — بعد تثبيت impeccable، صقل اللوحة (critique/onboard/animate)، واكتشاف API بسيطة الحقيقي، وبدء إعادة تصميم لاتجاه «بيان أصول» (يحتاج إرجاعه لألوان TIC).

## 1. هوية المشروع
- الجهة: **مدينة طايا الصناعية (TIC)** — لوحة قرار مالك تتابع صفقات السجل العقاري حول طايا بالرياض.
- المستخدم: أبو تركي · ab.alkhaled@gmail.com (git: meask562).
- الهدف: المدير يفتح **رابطاً واحداً** يرى فيه أين السوق وأين يقف منه ويقرّر خلال ثوانٍ.

## 2. الحالة الحاليّة
- **اللوحة مبنيّة بالكامل ومصقولة** (`site/index.html`): data-driven من `site/data/metrics.json` — KPI،
  «موقعك من السوق» (إدخال يدوي)، حجم السوق، سيولة/طلب (Chart.js)، خريطة Leaflet + فلاتر، اتجاه (تبديل)،
  مخاطر، جدول قابل للفرز/الفلترة. + حالات onboard (تحميل/خطأ/فارغ/تلميح أول مرة/شروح ⓘ) + حركة (عدّ
  تصاعدي + نبضة الفرق). **بيانات توضيحية** (sample=true) حتى أول جلب فعلي.
- **معمارية:** جلب محلي (Playwright + Chrome مسجَّل في بسيطة) → analytics → git push → Vercel.
- **مصدر الحقيقة:** `site/data/metrics.json` (يُولَّد من `fetcher/analytics.py`).
- **معاينة محليّة:** `cd site && python3 -m http.server 8000` ثم `http://localhost:8000`.

## 4. ما أُنجز في هذه الجلسة (2026-06-20)
- تثبيت **impeccable** + `PRODUCT.md`/`DESIGN.md`. إعادة تصميم اللوحة لمعيار impeccable.
- **critique** (31/40) → **onboard** (حالات أول مرة/فارغة/خطأ بلغة المدير) → **animate** (عدّ الأرقام).
- **اكتشاف API بسيطة الحقيقي** (المفترض كان خاطئاً 403): الصحيح
  `GET /api/get_geojson_filterd_rer_transactions?sw_lat=&sw_lng=&ne_lat=&ne_lng=` (GeoJSON). وملف
  Chrome `~/Library/Application Support/taya-chrome-profile` **مسجَّل دخوله في بسيطة فعلاً**. `.env` أُنشئ.
- 3 موكابات لإعادة تصميم جريئة؛ المالك اعتمد **C (بيان أصول)** — لكن **بشرط ألوان TIC**.
- إعدادات دائمة: `~/.claude/settings.json` (`defaultMode:bypassPermissions` + deny لأوامر خطرة).
